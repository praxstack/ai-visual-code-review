## 2024-03-03 - [Fix Command Injection Vulnerabilities in Node.js Execution]
**Vulnerability:** Execution of dynamic arguments in `child_process.execSync` allowed arbitrary command injection.
**Learning:** `execSync` executes commands via the shell, which evaluates metacharacters (e.g., `;`, `|`, `&&`) and allows an attacker to execute arbitrary commands if input is not properly sanitized.
**Prevention:** Use `child_process.execFileSync` or `spawnSync` with an array of arguments to execute commands directly without a shell, which naturally prevents command injection since arguments are passed verbatim to the executable.
## 2024-03-04 - [Fix DOM XSS in Frontend Template Interpolation]
**Vulnerability:** The `escapeHtml` function used DOM `.textContent` assignment which failed to escape single and double quotes. Interpolating these unescaped values into `onclick` attributes created a DOM-based XSS vulnerability.
**Learning:** Using `.innerHTML` / `.textContent` for HTML escaping is unsafe for attributes and inline JavaScript handlers, because quotes (`'` and `"`) are left intact. Furthermore, even properly escaping quotes is brittle when interpolating strings directly into inline JS handlers (like `onclick="fn('VAL')"`) because the HTML parser decodes entities (like `&#039;`) before executing the JS, thereby bypassing the escaping.
**Prevention:** Always use regex-based string replacement for `escapeHtml` (handling `&`, `<`, `>`, `"`, `'`). To securely pass dynamic strings to inline event handlers, inject the escaped value into a `data-*` attribute and retrieve it in the handler using `this.getAttribute('data-*')`, which bypasses the HTML-to-JS decoding vulnerability completely.
